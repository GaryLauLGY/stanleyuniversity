主题精华

今天值得沉淀的 4 条线

1. 排障先从环境变量外层开始

网络、代理、重启仍然是第一层检查，能快速排除最常见故障。

2. 网页可用不等于 CLI 可用

当网页端正常而 Codex 不可用时，要继续检查本机客户端、API 配置和代理链路差异。

3. API 切换工具需要边界感

ccswitch 这类工具能提升效率，但误改配置也可能带来更重的恢复成本。

4. 配置定位要形成清单

Windows 场景下 profile 和 config 是优先检查位置，可以沉淀成新人排障说明。
